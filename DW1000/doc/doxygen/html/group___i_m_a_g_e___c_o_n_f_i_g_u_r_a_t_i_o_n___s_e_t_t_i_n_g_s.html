#define dg_configBLACK_ORCA_MB_REV   BLACK_ORCA_MB_REV_D

The chip revision that we compile for.

Note
There is no default value defined for the target chip revision. The application must
neither explicitly set dg_configBLACK_ORCA_IC_REV, nor set
dg_configUSE_AUTO_CHIP_DETECTION to 1.

{,} The chip stepping that we compile for.

Note
There is no default value defined for the target chip revision. The application must
either explicitly set dg_configBLACK_ORCA_IC_STEP nor set
dg_configUSE_AUTO_CHIP_DETECTION to 1.

{,} The motherboard revision we compile for.

◆ dg_configSKIP_MAGIC_CHECK_AT_START

#define dg_configSKIP_MAGIC_CHECK_AT_START   (0)

When set to 1, the delay at the start of execution of the Reset_Handler is skipped.

This delay is added in order to facilitate proper programming of the Flash or QSPI
launcher invocation. Without it, there is no guarantee that the execution of the image
will not proceed, altering the default configuration of the system from the one that the
bootloader leaves it.

{, }

◆ dg_configUSE_AUTO_CHIP_DETECTION

#define dg_configUSE_AUTO_CHIP_DETECTION   (0)

When set to 1, the chip version (DA14680/1-01 or DA14682/3-BA ) will be detected
automatically.

Note
It cannot be used in BLE applications because of the different linker scripts
that are used.
Warning
Not to be used by a generic project, applicable for uartboot only.
