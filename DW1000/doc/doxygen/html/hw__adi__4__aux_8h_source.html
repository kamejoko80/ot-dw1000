344 // Register: ADI_4_AUX_O_ADC0
345 //
346 //*****************************************************************************
347 // Field: [7] SMPL_MODE
348 //
349 // ADC Sampling mode:
350 //
351 // 0: Synchronous mode
352 // 1: Asynchronous mode
353 //
354 // The ADC does a sample-and-hold before conversion. In synchronous mode the
355 // sampling starts when the ADC clock detects a rising edge on the trigger
356 // signal. Jitter/uncertainty will be inferred in the detection if the trigger
357 // signal originates from a domain that is asynchronous to the ADC clock.
358 // SMPL_CYCLE_EXP determines the the duration of sampling.
359 // Conversion starts immediately after sampling ends.
360 //
361 // In asynchronous mode the sampling is continuous when enabled. Sampling ends
362 // and conversion starts immediately with the rising edge of the trigger
363 // signal. Sampling restarts when the conversion has finished.
364 // Asynchronous mode is useful when it is important to avoid jitter in the
365 // sampling instant of an externally driven signal
366 #define ADI_4_AUX_ADC0_SMPL_MODE 0x00000080
367 #define ADI_4_AUX_ADC0_SMPL_MODE_M 0x00000080
368 #define ADI_4_AUX_ADC0_SMPL_MODE_S 7
369 
370 // Field: [6:3] SMPL_CYCLE_EXP
371 //
372 // Controls the sampling duration before conversion when the ADC is operated in
373 // synchronous mode (SMPL_MODE = 0). The setting has no effect in asynchronous
374 // mode. The sampling duration is given as 2^(SMPL_CYCLE_EXP + 1) / 6 us.
375 // ENUMs:
376 // 10P9_MS 65536x 6 MHz clock periods = 10.9ms
377 // 5P46_MS 32768x 6 MHz clock periods = 5.46ms
378 // 2P73_MS 16384x 6 MHz clock periods = 2.73ms
379 // 1P37_MS 8192x 6 MHz clock periods = 1.37ms
380 // 682_US 4096x 6 MHz clock periods = 682us
381 // 341_US 2048x 6 MHz clock periods = 341us

384 // 42P6_US 256x 6 MHz clock periods = 42.6us
385 // 21P3_US 128x 6 MHz clock periods = 21.3us
386 // 10P6_US 64x 6 MHz clock periods = 10.6us
387 // 5P3_US 32x 6 MHz clock periods = 5.3us
