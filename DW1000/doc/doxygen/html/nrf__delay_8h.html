__STATIC_INLINE void nrf_delay_ms ( uint32_t  number_of_ms)

Function for delaying execution for number of miliseconds.

Note
NRF52 has instruction cache and because of that delay is not precise.
Function internally calls nrf_delay_us so the maximum delay is the same as in case of nrf_delay_us, approx. 71 minutes.
