◆ nrf_drv_radio802154_auto_pending_bit_set()

void nrf_drv_radio802154_auto_pending_bit_set ( bool  enabled)

Enable or disable setting pending bit in automatically transmitted ACK frames.

addresses and Pan Id of this device.

Radio driver automatically sends ACK frames in response to unicast frames destined to this node. Pending bit in ACK frame can be set or cleared regarding data in pending buffer destined to ACK destination.

If setting pending bit in ACK frames is disabled, pending bit in every ACK frame is set. If setting pending bit in ACK frames is enabled, radio driver checks if there is data in pending buffer destined to ACK destination. If there is no such data, pending bit is cleared.

Note
It is possible that if there is a lot of supported peers radio driver cannot verify if there is pending data before ACK is sent. In this case pending bit is set.
Parameters
[in]enabledIf setting pending bit in ACK frames is enabled.

Change radio state to Energy Detection.

Note
This function should be called in Receive state. In other states energy detection procedure will be scheduled.

In Energy Detection state radio detects maximum energy for given time. Result of the detection is reported to the higher layer by nrf_radio802154_energy_detected() call.

Parameters
[in]channelChannel number on which radio will detect energy.
[in]time_usDuration of energy detection procedure. Given value is rounded up to multiplication of 10s (128 us).
Returns
true If the energy detection procedure was scheduled.
false If the driver could not schedule the energy detection procedure.

◆ nrf_drv_radio802154_transmit()

bool nrf_drv_radio802154_transmit ( const uint8_t *  p_data,
uint8_t  channel,
int8_t  power 
)

Change radio state to Transmit.

Note
This function should be called in Receive state. In other states transmission will be scheduled.

In Transmit state radio transmits given frame. If requested it waits for ACK frame. Radio driver wait infinitely for ACK frame. Higher layer is responsible to call nrf_radio802154_receive() after ACK timeout. Transmission result is reported to higher layer by nrf_radio802154_transmitted() or nrf_radio802154_busy_channel() calls.

Parameters
[in]p_dataPointer to array containing data to transmit. First byte should contain frame length and following bytes should contain data. CRC is computed automatically by radio hardware and can contain any bytes.
[in]channelChannel number on which radio will transmit given frame.
[in]powerTransmission power [dBm]. Given value is rounded up to nearest permitted value.
Returns
true If the transmission procedure was scheduled.
false If the driver could not schedule the transmission procedure.

◆ nrf_drv_radio802154_transmitted()

void nrf_drv_radio802154_transmitted ( uint8_t *  p_ack)

Notify that frame was transmitted.

Note
If ACK was requested for transmitted frame this function is called after proper ACK is received. If ACK was not requested this function is called just after transmission ends.
Buffer pointed by the p_ack pointer is not modified by the radio driver (and can't be used to receive a frame) until nrf_drv_radio802154_buffer_free() function is called.
Buffer pointed by the p_ack pointer may be modified by the function handler (and other modules) until nrf_drv_radio802154_buffer_free() function is called.
Parameters
[in]p_ackPointer to received ACK buffer. The first byte in the buffer is length of the frame and following bytes are the ACK frame itself (after PHR). If ACK was not requested p_ack is set to NULL.
